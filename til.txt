< git 강의 1주차 키워드 >
1. 버전 관리
> 프로젝트 상태가 변경되는 정보들을 관리하는 것. Git은 commit을 사용해서 버전이 달라지는 것을 관리한다.

2. git초기화
> 컴퓨터에 있는 프로젝트를 Git이 관리하는 프로젝트로 만드는 것을 Git 초기화 한다고 말한다.

3. commit
> 현재 프로젝트의 상태를 사진으로 찍어 저장하는 것을 말한다
> commit은 누가(author), 언제(when) commit 했는지의 정보와 프로젝트 변경 내용이 담겨있다.
> 작업 내역이 어떤 것인지 알아볼 수 있게 적는 메시지를 commit 메시지라고 한다.

3. pull
> 원격 repo의 commit들을 로컬 reop에 반영하는 것을 pull 이라고 한다.

4. push
> 로컬 repo의 commit들을 원격 reop에 반영하는 것을 push 라고 한다.

5. add( 혹은 staging): commit에 반영할 지 안할 지를 파일 단위로 선택하는 것.

6. commit history
> commit 한 시간 순서대로 리스트가 저장되어 있는 것.

7. repo
> Git으로 관리하는 프로젝트들이 있는 저장소라는 뜻. Repository

8. Tracking
> 로컬 repo와 원격 repo를 연결하는 것.

9. clone
> 원격 repo를 내 컴퓨터로 가져와서 초기 repo를 세팅하는 것을 말한다.
